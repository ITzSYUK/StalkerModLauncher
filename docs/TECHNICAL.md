# Техническая документация

## Назначение и совместимость

S.T.A.L.K.E.R. Mod Launcher работает с локальными папками игр и модификаций. Он не привязан к Steam и не использует облачные сервисы или базу данных.

Главный сценарий: базовая DRM-free установка одной из игр серии S.T.A.L.K.E.R. и набор папок модов, которые нужно наложить друг на друга. Файловая модель применима к Shadow of Chernobyl, Clear Sky, Call of Pripyat, Anomaly, OGSR и другим сборкам с типичной структурой X-Ray: `gamedata`, `fsgame.ltx`, `bin` или `bin_x64`, архивы `gamedata.db*`.

Отдельный игровой адаптер потребуется только для проекта с принципиально другой структурой файлов или правилами запуска.

## Профили

Профиль хранит имя, описание, путь к игре, список модов, их порядок и состояние, путь к выбранному EXE, параметры запуска и путь к workspace.

Есть два типа профилей:

- **Обычный профиль** объединяет базовую игру и включённые моды в отдельной рабочей среде.
- **Автономный мод** запускается прямо из своей корневой папки. Базовая игра и workspace ему не нужны.

Порядок включённых модов идёт сверху вниз: мод ниже в списке имеет больший приоритет и заменяет совпадающие файлы более раннего мода. Это ожидаемый способ ставить патчи и совместимые дополнения.

## Workspace

Для обычного профиля лаунчер создаёт управляемую папку вне установленной игры:

```text
<диск игры>\StalkerModLauncher\Workspaces\<имя профиля>-<короткий ID>\
  current\
  userdata\
  build-manifest.json
```

- `current` — восстановимый кэш с объединённым представлением игры и включённых модов;
- `userdata` — отдельные сохранения, `user.ltx`, логи и скриншоты профиля;
- `build-manifest.json` — подпись успешной сборки и статистика файлов.

Файлы на одном NTFS-диске подключаются через hard link, а файлы с других дисков — через symbolic link. Независимая копия создаётся для `fsgame.ltx`, потому что лаунчер переписывает его для изоляции `userdata`. Для read-only файлов используется symbolic link либо отдельная локальная копия. Если безопасный способ подключить файл недоступен, сборка останавливается с ошибкой вместо скрытого копирования больших объёмов данных.

Для symbolic link между разными дисками Windows может потребовать включить **Режим разработчика** или запустить лаунчер от имени администратора.

Проводник может показывать большой размер workspace, потому что считает размер hard-linked файлов как будто они лежат отдельно. Это не означает фактическое дублирование данных на диске.

## Безопасность файлов

Лаунчер не пишет в исходную папку игры и не удаляет папки модов. Удаление обычного профиля касается только его workspace, если он расположен в управляемой папке лаунчера и содержит защитный marker-файл. При удалении автономного профиля его исходная папка также остаётся на месте.

Marker-файл — это служебный файл `.stalker-launcher-workspace`, который лаунчер создаёт в собственном workspace. Перед очисткой, пересборкой или удалением он проверяет наличие этого файла и соответствие профилю. Если marker отсутствует, лаунчер отказывается удалять содержимое: это защита от случайного выбора и очистки чужой папки.

Перед запуском проверяются пути игры и модов, выбранный EXE, `fsgame.ltx`, свободное место, состояние workspace и возможность создать symbolic link. Ошибки блокируют запуск; предупреждения только сообщают о возможной проблеме.

## Настройки и перенос

Настройки хранятся в JSON:

```text
%AppData%\StalkerModLauncher\settings.json
%AppData%\StalkerModLauncher\settings.backup.json
```

Запись выполняется атомарно через временный файл и резервную копию. Одновременно работает только один экземпляр лаунчера, чтобы исключить конфликт сохранения.

Пути к игре и модам абсолютные. После ручного переноса игры или модов укажите новые папки в настройках профиля. Workspace не нужно переносить вручную: в окне **Состояние** есть команда **Перенести**. Она копирует `userdata` в новый управляемый workspace; `current` не переносится и будет собран заново.

## Выбор EXE

Лаунчер ищет известные X-Ray исполняемые файлы, включая `xrEngine.exe`, `XR_3DA.exe`, `AnomalyLauncher.exe` и другие `.exe` в типичных каталогах `bin` и `bin_x64`. При автоматическом выборе более поздний включённый мод имеет приоритет перед базовой игрой. В настройках профиля можно закрепить EXE вручную; в этом случае автоматический выбор его не заменяет.

## Диагностика

Окно **Состояние** показывает готовность профиля, источники файлов, выбранный EXE, состояние workspace, сохранения, последний игровой лог и crash dump. Там же доступны:

- **Пересобрать** — заново подготовить `current` без запуска игры;
- **Очистить кэш** — удалить только `current`, не затрагивая `userdata`;
- **Перенести** — безопасно перенести профильные данные в новый workspace;
- **Профиль**, **Сохранения**, **Последний лог**, **Crash dump** — открыть связанные файлы в Проводнике.

Статистика workspace берётся из manifest, поэтому открытие окна не требует полного обхода файлов. Для старых workspace подробные размеры появятся после пересборки.

## Браузер модификаций AP-PRO

Браузер модификаций получает публичные страницы категорий AP-PRO для Тени Чернобыля, Чистого Неба и Зова Припяти. Для карточки извлекаются название, краткое описание, URL обложки, оценка, просмотры и ссылка на исходную страницу.

Первая страница выбранной категории загружается при открытии или переключении раздела. Количество страниц определяется по `data-pages` в HTML пагинации сайта. Следующие страницы запрашиваются только когда пользователь прокручивает список близко к низу, поэтому каталог отображается одним непрерывным списком без элементов пагинации. Карточки с одинаковым URL не добавляются повторно.

Данные раздела и загруженные обложки хранятся только в памяти окна; ответы каталога кэшируются примерно на 10 минут. Нужен интернет. Лаунчер использует обычный HTTP-запрос и не обходит защиту AP-PRO: если сайт выдаёт проверку браузера или запрос не проходит, пользователь увидит сообщение об ошибке и сможет открыть [AP-PRO.RU](https://ap-pro.ru/) вручную.

Браузер не загружает файлы модов, не распаковывает архивы, не изменяет профиль и не устанавливает моды.

## Скриншоты и Discord

Окно **Скриншоты** ищет `.png`, `.jpg`, `.jpeg` и `.bmp` в каталогах профиля или автономной сборки. Скриншот можно скопировать через контекстное меню или двойным щелчком в полноэкранном режиме. Миниатюры и полноразмерные изображения освобождаются при закрытии окна.

Discord Rich Presence необязателен. Если для профиля включён статус и настроен Discord client ID, лаунчер показывает название активного профиля и состояние запуска. Детальные игровые данные, например локация или квест, могут отображаться только если сам движок или мод передаёт их в Discord.

## Сборка и публикация

Для разработки нужен .NET 8 SDK:

```powershell
dotnet build .\StalkerModLauncher.sln
dotnet test .\StalkerModLauncher.sln -c Release
dotnet run --project .\src\StalkerModLauncher\StalkerModLauncher.csproj
```

Текущая локальная публикация в `publish\stabilized` является single-file framework-dependent сборкой и использует установленный .NET 8 Desktop Runtime:

```powershell
dotnet publish .\src\StalkerModLauncher\StalkerModLauncher.csproj `
  -c Release `
  -r win-x64 `
  --self-contained false `
  -p:PublishSingleFile=true `
  -p:IncludeNativeLibrariesForSelfExtract=true `
  -p:DebugType=None `
  -p:DebugSymbols=false `
  -o .\publish\stabilized
```

Для самодостаточной публикации замените `--self-contained false` на `--self-contained true`.
